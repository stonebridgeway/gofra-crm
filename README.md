# ГОФРА CRM

Фронтенд-прототип CRM для работы с клиентами, сделками, контактами и историей
взаимодействий. Основные воронки представлены в формате канбан, а все точные
статусы из исходной таблицы доступны в карточках, фильтрах и справочниках.

## Что реализовано

- канбан и табличное представление клиентов;
- отдельная воронка сделок;
- контакты и история взаимодействий;
- кабинеты руководителя и сотрудника с разными срезами данных;
- интерактивный календарь, задачи, напоминания и перенос сроков;
- статистика по воронке, продажам, активности и команде с детализацией;
- внутренний демо-чат с контекстом клиентов;
- светлая, тёмная и системная темы;
- импорт лидов с сопоставлением полей и предпросмотром;
- редактируемые справочники;
- глобальный поиск, фильтры, карточки сущностей и мобильная навигация;
- локальное сохранение демоданных и автоматическая миграция схемы.

Это чистый frontend: сетевых запросов и бизнес-логики backend в прототипе нет.
Точка интеграции изолирована интерфейсом `CrmGateway`, поэтому локальное
хранилище можно заменить HTTP-адаптером без переделки экранов. Переключение
ролей служит только для демонстрации интерфейсов; настоящие права и авторизацию
должен проверять backend.

## Запуск

Требуется Node.js `>=22.13.0`.

```bash
pnpm install
pnpm run dev
```

Проверка перед публикацией:

```bash
pnpm run lint
pnpm run build
pnpm test
```

## GitHub Pages

Публикация выполняется вручную через workflow `Deploy GitHub Pages`. Статическая
сборка автоматически учитывает имя репозитория, поэтому JS, CSS, шрифты и
изображения корректно работают на project Pages без backend-сервера.
